Unlike what we did during lecture, we will be building a character level RNN. That is, we will treat each character as a token in our sequence, rather than each word.

Identify two advantage and two disadvantage of modelling SMS text messages as a sequence of characters rather than a sequence of words.

Answer:
advantage:
1) There are a lot of short forms of spelling (mispelling, custom word). For instance, people use "K" instead of "OK", "n" instead of "and", "wkly" instead of "weekly". There are also some informal words like "lol", "Nah". Overall, the text in SMS text messages is very different from texts from sources of wikipedia, literature, newspapeer. The word embedding tools like GloVe probably cannot recognize a lot of words in SMS text messages and would not work well. Working at a lower level of character is more reasonable. Otherwise, we would need to build our own word embedding model and tune for finding the best dimension, which takes a lot of effort and requires much more data than the size of current dataset.

2) The number of possible characters is much smaller than the number of distinct words. Moreover, the words in a SMS text are usually short or spelled in short-form. The embedding space required for character level is much more simpler, which indeed allows us to use one-hot encoding while Google Colab can still handle the memory space and computation quite easily.

 disadvantage:
1) RNN needs to be deeper when using characters, to account for the learning required for recognizing multiple consecutive characters as a word.

2) there is loss of word structural information when using characters. For example, "i use pen" becomes "i u s e p e n", and we lose information of when a word start and when a word end.

Answer:
When having a balanced training set, the model will focus more on learning the features that truly differ spam messages from non-spam messages, instead of classifying a sample as the major class. For instance, if there are only 1% of the SMS messages are spam, then a model has learnt nothing but simply assigning all mesages as non-spam would have a 99% accuracy. We do not want the model to have preference of one class over the other, but to really identify the difference.

Part 2. Model Building [5 pt]¶

Build a recurrent neural network model, using an architecture of your choosing. Use the one-hot embedding of each character as input to your recurrent network. Use one or more fully-connected layers to make the prediction based on your recurrent network output.

Instead of using the RNN output value for the final token, another often used strategy is to max-pool over the entire output array. That is, instead of calling something like:

out, _ = self.rnn(x)
self.fc(out[:, -1, :])

where self.rnn is an nn.RNN, nn.GRU, or nn.LSTM module, and self.fc is a fully-connected layer, we use:

out, _ = self.rnn(x)
self.fc(torch.max(out, dim=1)[0])

This works reasonably in practice. An even better alternative is to concatenate the max-pooling and average-pooling of the RNN outputs:

out, _ = self.rnn(x)
out = torch.cat([torch.max(out, dim=1)[0], 
                 torch.mean(out, dim=1)], dim=1)
self.fc(out)

We encourage you to try out all these options. The way you pool the RNN outputs is one of the "hyperparameters" that you can choose to tune later on.

class SMSRNN(nn.Module):
    def __init__(self, input_size, hidden_size, num_classes):
        super(SMSRNN, self).__init__()
        self.hidden_size = hidden_size
        self.rnn = nn.GRU(input_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, num_classes)
    
    def forward(self, x):
        # Set an initial hidden state
        h0 = torch.zeros(1, x.size(0), self.hidden_size)
        # Forward propagate the RNN
        out, _ = self.rnn(x, h0)
        # Pass the output of the last time step to the classifier
        out = self.fc(out[:, -1, :])
        return out

def get_accuracy(model, data):
    """ Compute the accuracy of the `model` across a dataset `data`
    
    Example usage:
    
    >>> model = MyRNN() # to be defined
    >>> get_accuracy(model, valid) # the variable `valid` is from above
    """
    correct, total = 0, 0
    indent = torch.eye(114)
    for batch in data:
        texts = batch.sms[0]
        texts = indent[texts]
        labels = batch.label
        output = model(texts)
        pred = output.max(1, keepdim=True)[1]
        correct += pred.eq(labels.view_as(pred)).sum().item()
        total += labels.shape[0]
    return correct / total

class SMSRNN_dropout(nn.Module):
    def __init__(self, input_size, hidden_size, num_classes):
        super(SMSRNN_dropout, self).__init__()
        self.hidden_size = hidden_size
        self.rnn = nn.GRU(input_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, num_classes)
        self.dropout = nn.Dropout(0.1)

    def forward(self, x):
        # Set an initial hidden state
        h0 = torch.zeros(1, x.size(0), self.hidden_size)
        # Forward propagate the RNN
        out, _ = self.rnn(x, h0)
        # Pass the output of the last time step to the classifier
        out = self.fc(self.dropout(out[:, -1, :]))

        return out

By setting pretrained = True we say to fastai to download the weights from the trained model (a corpus of 103 MM of wikipedia articles).
